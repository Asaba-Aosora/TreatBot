# 地理计算逻辑说明

## 问题：为什么"辽宁沈阳"匹配到"辽宁沈阳"距离是66km？

### 根本原因

坐标库中存在**多个坐标条目**，且字段名称不一致：

```python
LOCATION_COORDS = {
    '沈阳': (41.8057, 123.4328),      # 沈阳市市中心坐标
    '辽宁省': (41.2959, 123.1238),    # 辽宁省城（沈阳）坐标，但有偏差
}
```

**计算过程：**
1. 患者输入："辽宁沈阳" → 匹配到 `沈阳` → 坐标 (41.8057, 123.4328)
2. 试验记录："研究中心所在省份=辽宁省，研究中心所在城市=沈阳市"
3. `compute_geo_distance()` 优先使用 city：`find_location_coord('沈阳市')`
4. `find_location_coord` 通过**模糊匹配**找到 `沈阳` 条目
5. 但如果city里没有完全匹配的，就用province `辽宁省` → (41.2959, 123.1238)
6. **Haversine距离** = 约66km

### 改进方案

#### 1. **统一坐标表示**
- 移除重复条目
- 使用标准化名称（带"市"、"省"后缀）
- 保留一个准确的主坐标

#### 2. **多地点支持**
改进后的 `compute_geo_distance()` 现在：
- 将city和province中的**多个地点（用逗号分隔）**都计算距离
- 返回**最短距离**而不是单一距离

```python
# 支持多地点试验
city = "沈阳市,北京市,上海市"
# 计算患者到三个城市的距离，返回最小值
```

#### 3. **HTML高亮最近地点**
新增函数 `find_nearest_location()`：
- 返回最近的具体地点及其距离
- HTML渲染时高亮这个地点（绿色背景）
- 同一试验有多个地点时，只高亮最近的

### 技术实现

**新增/修改的函数：**

1. **`compute_geo_distance()`** - 计算最短距离
   - 支持多地点（逗号分隔）
   - 返回最小值

2. **`find_nearest_location()`** - 找最近地点
   - 返回 `{'location': '地点名', 'distance': 距离, 'type': 'city/province'}`
   - 用于HTML高亮

3. **`match_trial()` 更新**
   - 添加 `nearest_location` 字段到返回结果
   - 传递给HTML渲染函数

4. **HTML渲染更新**
   - 解析多地点并高亮最近的
   - 在两处高亮：摘要行和详细信息表

### 示例

输入：患者位置 = "辽宁沈阳"
试验1：沈阳市，北京市，上海市 → 高亮"沈阳市"（距离约0km）
试验2：长春市，济南市 → 高亮"长春市"（距离约400km）

##距离计算公式

使用 **Haversine公式**（大圆距离）：

$$d = 2R \cdot \arcsin\left(\sqrt{\sin^2\left(\frac{\Delta\phi}{2}\right) + \cos(\phi_1) \cos(\phi_2) \sin^2\left(\frac{\Delta\lambda}{2}\right)}\right)$$

其中：
- $R = 6371$ km（地球平均半径）
- $\phi$ = 纬度，$\lambda$ = 经度
- 返回单位：km

### 距离等级

| 距离 | geo_rank | 评分 | 说明 |
|------|----------|------|------|
| < 50 km | 0 | +8 | 极佳（同城） |
| 50-200 km | 1 | +5 | 很好（相邻省市） |
| > 200 km | 2 | +2 | 可以（较远） |
| 无坐标 | 1/2 | +5/+2 | 文本匹配 |
